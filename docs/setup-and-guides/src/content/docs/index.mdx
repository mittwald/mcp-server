## How an agent-driven Mittwald task works

<div class="home-flow">
	<div class="home-step">
		<div class="step-id">1. Intent</div>
		<p>You prompt the outcome you want on Mittwald.</p>
	</div>
	<div class="home-step">
		<div class="step-id">2. Discovery</div>
		<p>Your agent discovers relevant MCP tools and required inputs.</p>
	</div>
	<div class="home-step">
		<div class="step-id">3. Auth</div>
		<p>OAuth or token auth is used for each tool call.</p>
	</div>
	<div class="home-step">
		<div class="step-id">4. Execution</div>
		<p>The agent performs tool calls and reports results and IDs.</p>
	</div>
	<div class="home-step">
		<div class="step-id">5. Verify</div>
		<p>You run read checks and decide whether to proceed, retry, or roll back.</p>
	</div>
</div>

<p class="home-note"><strong>Human responsibility:</strong> destructive or cost-impacting changes should be approved explicitly. Treat the agent as an operator, not as an unattended process.</p>
