# Risk Register (Working Draft)

Severity: H/M/L. Status: Open/Triage/Mitigated.

- [ ] **Placeholder secrets in configs** (M, Open) — `.env.example` and `fly.toml` in `mittwald-oauth` ship hardcoded JWT/ENCRYPTION/SESSION/ADMIN tokens; risk of accidental use in lower envs or misconfigured prod. Mitigation: enforce secret injection via env/secret manager; add startup guard against placeholder values.
- [ ] **CORS wildcard in mittwald-oauth** (M, Open) — `.env.example` sets `CORS_ORIGIN=*`; if mirrored in prod, enables token exposure to any origin. Mitigation: enforce explicit allowlist per env; add runtime validation.
- [ ] **HS256 JWT signing defaults** (M, Open) — `mittwald-oauth` defaults to HS256 with static secret; need key rotation and strong entropy; assess if public clients interact. Mitigation: consider asymmetric keys; enforce length checks; add rotation mechanism and kid tracking.
- [ ] **Global CLI + openssh in MCP images** (L, Open) — Production images install `openssh-client` and global `@mittwald/cli`; increases surface and cache size. Mitigation: verify necessity; drop or scope to minimal runtime.
- [ ] **Placeholder Fly env values** (M, Open) — `packages/mcp-server/fly.toml` and `packages/oauth-bridge/fly.toml` include placeholder signing keys/base URLs; risk if deployed without overrides. Mitigation: startup checks for placeholder markers; CI policy to block default secrets.
- [ ] **DCR registration access token not validated** (H, Open) — `mittwald-oauth` `ClientValidator.validateRegistrationAccessToken` returns true for any token length ≥20 and does not bind to client or expiry, allowing unauthenticated read/update/delete of registered clients if access token is provided. Mitigation: persist and verify registration access tokens (JWT or DB), bind to client, enforce expiry, and reject absent tokens on GET/DELETE where required by RFC 7591.
- [ ] **OAuth state not one-time in MCP server** (M, Triage) — `src/auth/oauth-state-manager.ts` allows state reuse: `getState` does not delete and state is only cleared manually; codeVerifier/codeChallenge default to empty strings. Risk of replay if downstream handlers do not enforce single-use. Mitigation: delete on first read or mark consumed; store PKCE values early; add tests for replay rejection.
- [ ] **Bridge refresh tokens without refresh endpoint** (L, Open) — OAuth bridge mints refresh tokens but exposes no refresh grant route, causing unusable refresh tokens and potential confusion if clients rely on refresh. Mitigation: add refresh flow or remove refresh issuance to avoid dead tokens.
- [ ] **Redis/session/key TTL validation** (TBD, Triage) — Need to confirm single-use state tokens and TTLs in MCP bridge and server to prevent replay; pending code review.
- [ ] **Redirect allowlist enforcement** (TBD, Triage) — `BRIDGE_REDIRECT_URIS` optional; need to confirm default behavior (open or deny) to avoid open redirect in DCR/authorize. Pending code review.
- [ ] **Express 5 upgrade** (L, Open) — MCP uses Express 5 RC; verify middleware error handling and TypeScript types to avoid dropped errors or 404s. Mitigation: targeted tests and handler audits.
- [ ] **Shell exec for CLI** (M, Triage) — MCP CLI wrapper uses `child_process.exec` with manual quoting; relies on string concatenation and may be brittle if args contain tricky characters. Mitigation: switch to `spawn`/`execFile` with argv array; add tests for injection attempts and enforce allowlists per tool.
