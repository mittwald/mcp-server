## Report Generation Process

### Step 1: Load Data


### Step 2: Analyze Patterns

- Identify problem clusters
- Find domain-specific trends
- Correlate with tier classification
- Detect recurring failures

### Step 3: Generate Recommendations

Based on patterns:
- If auth errors > 5%: Check scope configuration
- If timeout errors > 10%: Review timeout settings
- If resource_not_found > 15%: Review dependency chain

### Step 4: Format Report

Use markdown formatting with:
- Clear section hierarchy
- Data tables for metrics
- Bullet points for recommendations
- Appendices for details
