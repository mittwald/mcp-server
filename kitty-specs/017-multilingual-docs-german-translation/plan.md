# Implementation Plan: Multilingual Documentation with German Translation

**Branch**: `017-multilingual-docs-german-translation` | **Date**: 2026-01-24 | **Spec**: [spec.md](spec.md)

## Summary

Add German language support to both Mittwald MCP documentation sites using Starlight's native i18n capabilities. The implementation configures i18n on both sites (Setup & Guides, Reference), then translates all 202 content files to German using AI-assisted translation.

**Approach**: Site-by-site translation
1. Configure i18n on both sites (shared configuration pattern)
2. Translate Setup & Guides site (7 files)
3. Translate Reference site (195 files, organized by domain)

## Technical Context

**Framework**: Astro Starlight 0.37.3 with built-in i18n support
**Languages**: English (root locale, no prefix), German (`/de/` prefix)
**Content Format**: Markdown/MDX with YAML frontmatter
**Build Tool**: Astro 5.6.1
**Testing**: Playwright for browser language detection and UI verification
**Target Platform**: Static site deployment (GitHub Pages, Fly.io, or custom)

**i18n Features Used**:
- Starlight `locales` configuration
- Root locale pattern (English without `/en/` prefix)
- Built-in language switcher widget
- Pre-built German UI strings
- Astro i18n routing for browser detection

## Implementation Phases

### Phase 1: i18n Configuration (Both Sites)

Update `astro.config.mjs` for both sites with identical i18n configuration:

```javascript
starlight({
  title: {
    en: 'Mittwald MCP',
    de: 'Mittwald MCP'
  },
  locales: {
    root: {
      label: 'English',
      lang: 'en'
    },
    de: {
      label: 'Deutsch',
      lang: 'de'
    }
  },
  defaultLocale: 'root',
  // ... existing sidebar configuration unchanged
})
```

**Verification**: Build both sites, verify language switcher appears, verify English URLs unchanged.

### Phase 2: Setup & Guides Translation (7 files)

Translate in order:
1. `index.mdx` - Landing page
2. `getting-started/index.md` - Getting started overview

4. `getting-started/codex-cli.md` - Codex CLI setup
5. `explainers/what-is-mcp.md` - MCP explanation
6. `explainers/what-is-agentic-coding.md` - Agentic coding explanation
7. `explainers/oauth-integration.md` - OAuth integration explanation

**Translation Rules**:
- Translate frontmatter: title, description, sidebar.label
- Translate all prose content
- Keep code examples unchanged (English)
- Keep technical terms (MCP, OAuth, PKCE) in English
- Use professional German technical writing style
